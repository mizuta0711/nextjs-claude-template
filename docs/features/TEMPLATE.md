## 運用ガイド

<!-- 
この設計書の運用フロー（作成後、このセクションは削除してよい）:

### 設計フェーズ
1. このテンプレートをコピーして `docs/features/yyyymmdd_機能名.md` を作成
2. メタ情報のステータスを「🔵 設計中」に設定
3. タスク一覧を全て「🔵 未実施」で作成
4. 設計詳細をフェーズごとに記述
5. 「4. ブラウザ評価計画」を記入する
6. /design-review で設計レビュー（評価計画のレビューも含む）を実施

### 実装フェーズ（フェーズごとに繰り返す）
1. メタ情報のステータスを「🟡 実装中」に更新
2. coding-specialist でフェーズの実装を実施
3. 実装完了したタスクのステータスを「✅ 完了」に更新
4. 保留・却下を決定したタスクのステータスと理由を更新
5. code-reviewer でコードレビューを実施
6. /browser-test でスポットテストを実施（評価計画に基づく。コードレビュー後の最終形に対して実行）
7. npm run build でビルド確認
8. docs/設計書/ 配下の関連ドキュメントを更新
9. git commit（フェーズ単位）

### 完了時
1. 全タスクのステータスが最終状態（✅/⏸️/❌）であることを確認
2. メタ情報のステータスを「🟢 完了」または「⏸️ 一部保留」に更新
3. 完了 → docs/features/completed/ に移動
4. 一部保留 → そのまま docs/features/ に残す（保留タスクの対応時に再開）
-->
