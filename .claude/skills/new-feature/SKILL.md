# 新規機能設計書の作成

## 手順

1. テンプレートを読み込む: `docs/features/TEMPLATE.md`
2. 以下の命名規則でファイルを作成:
   - パス: `docs/features/yyyymmdd_$ARGUMENTS.md`
   - 日付は本日の日付（yyyy-mm-dd形式ではなく yyyymmdd形式）
   - 機能名は引数で指定されたもの

3. テンプレートの以下のプレースホルダを埋める:
   - `{{機能名}}` → $ARGUMENTS
   - `{{日付}}` → 本日の日付
   - `{{ステータス}}` → `🔵 計画中`
   - `{{作成者}}` → `Claude Code`

4. ファイル作成後、ユーザーに以下を報告:
   - 作成したファイルパス
   - テンプレートの構造（セクション一覧）
   - 「設計内容を記入してください」のメッセージ
   - 💡 「`4. ブラウザ評価計画` も記入してください。`/design-review` 実行時に評価計画のレビューも行います」の案内

## テンプレートが見つからない場合

`docs/features/TEMPLATE.md` が存在しない場合は、最低限の構造で作成:

```markdown
# $ARGUMENTS

## メタ情報
| 項目 | 内容 |
|------|------|
| ステータス | 🔵 計画中 |
| 作成日 | {本日の日付} |
| 最終更新 | {本日の日付} |
| 作成者 | Claude Code |

## 概要
（機能の目的・背景を記載）

## タスク一覧
| # | タスク | ステータス | 備考 |
|---|--------|-----------|------|
| 1 | （タスク名） | 🔵 未実施 | |

## 設計書の更新計画
（実装時に更新が必要な docs/設計書/ のファイルを記載）

## 改訂履歴
| 版数 | 日付 | 内容 | 担当 |
|------|------|------|------|
| 1.0 | {本日の日付} | 初版作成 | Claude Code |
```
